Merkle-Hellman

This online help describes the functionality of the plug-in Merkle-Hellman.

The plug-in can be started via the menu Visualizations or via the crypto explorer tab Visualizations.

The plug-in consists of a description field and three basic areas: Key generation, Encryption and Decryption.

In the description field the corresponding statements about the algorithm in its sequence are displayed dynamically, depending on which basic step the user is located.

1) Key generation: This section summarizes the actions that are required to generate a private key and a public key.

First of all the number of the elements for the private key and the initial value must be selected from the drop-down boxes. By default, these values are ​​4 and 32. On this basis, the user can enter the values ​​of A (i), M and W manually using automatically and randomly generated sample values ​​or allow new values ​​to generate private key by clicking on the button Generate private key

By clicking on the button Generate public key, the public key elements are (B (i) s) generated and displayed under Public key B. Now, after the public key was also calculated, you can continue with the encryption.

2) Encryption: In the second section, the user can enter the message m in the corresponding field. As you type the message, the binary representation is dynamically calculated and shown below in addition to the binary representation of m. Once a valid value is entered by m, the button Encrypt is activated.

When you click on the button the message parts are calculated and entered into the calculations of the iterations in the underlying table. The ciphertext is also calculated, displayed and entered in the decryption.
