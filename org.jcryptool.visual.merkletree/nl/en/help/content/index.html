1) Merkle Signature Scheme (MSS)

Introduction

Using the Merkle Signature Scheme, messages can be signed quantum computer resistant. The signature consists out of two parts: the one-time signature (OTS) and the MerkleTree (a binary hashtree). The advantage of MerkleTrees is, that only one Merkle public key is needed for several OTS public keys. This is implemented by summarizing the hash values of the public key using the OTS MerkleTrees into a single hash value at the top of the tree. This value in the root node contains the Merkle public key.

2) eXtended Merkle Signature Scheme (XMSS)

Introduction

The eXtended Merkle Signature Scheme (XMSS) is a method based on the Merkle Signature Scheme. The difference of to the MSS is the usage of bitmasks and the L-Trees. With this extensions no collision-resistent hashfunction is needed. Only a second preimage resistant hashfunction is needed to guarantee security. Because only a second preimage resistant hashfunction is needed, the signature will get smaller, than using the MSS. The eXtended Merkle Signature Scheme uses as OTS the Winternitz One Time Signature (WOTS). There are two different variations, WOTS+ or WOTS$. In this plugin we use the WOTS+ scheme. In both variations, there is a freely chosen number of WOTS keypairs generated out of a random seed with the help of a pseudo-random key generator.

Bitmaske

The bitmask is a pseudo-random binary value. The bitmask is used to avoid the need of a collision resistant hashfunction. Bitmasks are used in the XMSS-Tree and in the L-Tree.

Winternitz Parameter

The winternitz parameter is used in the Winternitz-OTS (W-OTS). It determines the signature length. A larger value leads to shorter signatures but increased computation effort. It does not affect the scheme's security.

L-Tree

L-Trees are using the same hashfunctions as the XMSS-Tree, but a different bitmask. Every L-Tree has the same bitmask.

3) Usage

Tab: Seed and Key Generation

In this tab you can switch between the different modes. There is also an option to define the seed,set the winternitz parameter, set the amount of the secret keys and generate the secret keys.

This image shows where the seed is defined. The seed is used to generate the secret keys. It is possible to use an own defined numeric seed, or create a random seed with the button Generate new Seed.

If the eXtended-Merkle-Signature-Scheme (XMSS) is used, there is another box, where you can define the seed for the bitmask.

The winternitz parameter can be set to either 4 or 16 as these are the most commonly used values.

Tab: MerkleTree

This tab is used to display the created Merkle tree. The tree consists of leafs, the nodes and the root node. The root node is also known as the Merkle public key. If a node is chosen, the path to the root node is marked yellow. The verification path is marked red. If a node or a leaf is chosen the hash of this node is displayed. The full tree can be view by clicking Beschreibung ausblenden.

Tab: Key Pair

The public key and the private key are shown here. You can highlight the respective key parts by clicking the buttons.

Tab: Signing

This tab is used to generate signatures. The signature are generated using the keys, defined in tab 1. You can enter a user defined text and afte pressing Sign Message this message is going to be signed, using the merkle signature scheme.

This picture shows a signature. The Signature of the Message has the following content: Index of the Keypair | OTS-Signature | Authentication Path.

Tab: Verification

The signature can be verified, when the correct index is chosen. This one (???) is chosen initially. The verification starts by calculating the WOTS public key of the WOTS signature. This WOTS key is used to calculate the corresponding leaf of the L-Tree.

You can also choose another (wrong) index. If you do so, the verification will fail.
