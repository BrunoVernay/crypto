Certificate Verification

Welcome to the online help of the plug-in which validates certificates according to the validity models:

  • shell model
  • modified shell model
  • and chain model.

Below you find an overview of the plug-in and a more detailed description of the usage of the plug-in. This plugin helps to demonstrate the validity of digital signatures using different certificate trees, consisting of a Root CA, CA and a User CA, and validity models. It's also possible to change the period of validity of each certificate as well as the verification and signature date with sliders. After this modification you can validate this information. But you have to consider that these changes are just a simulation. That means: No certificates were changed and there were also no new certificates created.

General information

Reset function

Everything is reset to the initial state of the plug-in.

PKI Plugin

This plug-in builds on top of the PKI plug-in. The button starts the PKI plug-in.

Signature Verification

This button starts the Signature Verification plug-in.

 

Usage of the plug-in

Below you find how to use the plug-in on the basis of an example scenario.

Step 1

The first step is to load the certificates. If you click on the buttons "Load Root CA", "Load CA" and "Load User Certificate" a dialog window opens and you can choose the certificate which you want to load out of the JCT-Keystore. After you have chosen the right certificate you have to click on "load" and then on "finish". The sliders in the Graphical User Interface are now in the right position and the font colour of the buttons is now green.

 

Step 2

If all three certificates (Root CA, CA and User CA) were loaded successfully you can choose the validity model with the radio buttons. Now you can press the "Validate" button. If the symbol next to this button is an X in a red circle the validation was unsuccessful if the symbol is a checkmark in a green circle the validation was successful.

The log shows you all the information, which leads to this result.

Step:

There is the possibility to change the position of the sliders to simulate what would happen if the dates change. But you have to consider that these changes are just a simulation. That means: No certificates were changed and there were also no new certificates created. The verification slider can only be manipulated in the shell model. It's possible to adjust the months of a date with the sliders. If you want to set the days too you can do this with the text fields in "Details". After the modification of a slider the font colour of the buttons "Load Root CA", "Load CA" and "Load User Certificate" changes to orange.
